# shadcn/ui Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install shadcn/ui and migrate all hand-rolled button, input, and badge patterns to shadcn primitives while preserving the existing purple design system.

**Architecture:** Run `npx shadcn@latest init` to install dependencies and scaffold `components.json`, then immediately replace `index.css` with a custom version that layers our existing design tokens over shadcn's CSS variable format. Add `button`, `input`, and `badge` components, then migrate each component file to use them.

**Tech Stack:** shadcn/ui (latest), Radix UI, class-variance-authority, lucide-react, Tailwind v4, React 19, TypeScript

---

**Created:**

**Unchanged (intentionally):**

- Difficulty filter buttons (All/Easy/Medium/Hard) — custom color-specific active states, keep raw

- The mobile toggle in ProblemView — layout-critical, too custom
- `<textarea>` in ChatView — specialized with rows/resize, not worth abstracting
- Difficulty `<span>` color badges — these are just colored text, not semantic badges

### NavBar

- [ ] **Step 3: Replace `frontend/src/components/NavBar.tsx`**

The nav toggle buttons use `variant="secondary"` when active and `variant="ghost"` when inactive — this matches the existing `bg-secondary` / transparent pattern exactly.

```tsx
import type { Session } from '@supabase/supabase-js'
import { supabase } from '../lib/supabase'
import { Button } from './ui/button'

type View = 'practice' | 'search'

interface Props {
  view: View
  onNavigate: (v: View) => void
  session: Session | null
  authLoading: boolean
}

export function NavBar({ view, onNavigate, session, authLoading }: Props) {
  const handleSignIn = async () => {
    await supabase.auth.signInWithOAuth({
      provider: 'google',
      options: { redirectTo: window.location.origin },
    })
  }

  const handleSignOut = async () => {
    await supabase.auth.signOut()
  }

  return (
    <div className="flex items-center gap-1 px-4 py-2 border-b border-border bg-background shrink-0">
      {(['practice', 'search'] as View[]).map(v => (
        <Button
          key={v}
          variant={view === v ? 'secondary' : 'ghost'}
          size="sm"
          onClick={() => onNavigate(v)}
        >
          {v.charAt(0).toUpperCase() + v.slice(1)}
        </Button>
      ))}

      <div className="ml-auto">
        {authLoading ? null : session ? (
          <Button variant="ghost" size="sm" onClick={() => void handleSignOut()}>
            Sign out
          </Button>
        ) : (
          <Button size="sm" onClick={() => void handleSignIn()}>
            Sign in
          </Button>
        )}
      </div>
    </div>
  )
}
```
